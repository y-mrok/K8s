# Manual Scheduling
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx
  labels:
    name: nginx
spec:
  containers:
  - name: nginx
    image: nginx
    ports:
      - containerPort: 8080
```
- すべての Pod にはデフォルトでは表示されない `nodeName` というフィールドがある

## nodeName の指定がない場合の Pod のスケジューリング手順
1. スケジューラはすべての Pod の定義ファイルを調べ、`nodeName` の設定がないものをスケジューリング候補としてピックアップ
2. スケジューリングアルゴリズムにもとづき、ピックアップした Pod を配置する適切なノードを決定
3. 配置先のノード名を配置する Pod の `nodeName` に設定
4. ノードの Pod をスケジューリング 
- スケジューラがない場合は Pod の STATUS は Pending （保留状態）になる

## スケジューラがない場合の配置先ノードの指定
### Pod が未作成の場合
`nodeName` を使用し、配置先のノードを指定することで、Pod を作成するときに指定したノードに配置される
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx
  labels:
    name: nginx
spec:
  containers:
  - name: nginx
    image: nginx
    ports:
      - containerPort: 8080
  nodeName: node02
```
### Pod がノードに割当済みの場合
ノードに配置済みの Pod を別のノードに割り当てる場合 `nodeName` は使用できない。 Binding オブジェクトを作成し、 Pod Binding API に JSON 形式でリクエスト送信する。
```yaml
apiVersion: v1
kind: Binding
metadata:
  name: nginx
target:
  apiversion: v1
  kind: Node
  name: node02
```
上記定義ファイルを JSON 形式に変換して Pod Binding API に投げる
```
curl --header "Context-Type:application/json" --request POST --data ("apiVersion":"v1", "kind":"Binding"... ) http://$SERVER/api/v1/namespaces/default/pods/$PODNAME/binding/
```

# ラベルとセレクター
## ラベルとセレクター
- ラベル  
  オブジェクトをフィルタリングするためのプロパティ
- セレクター  
  条件を指定してオブジェクトをフィルタリングする機能
## ラベル
- 定義ファイルの `metadata` 内の `labels` セクション内にキーバリュー形式で定義する
- ラベルは複数設定できる
  ```yaml
  apiVersion: v1
  kind: Pod
  metadata:
    name: simple-webapp
    labels:
      app: App1
      function: Front-End
  spec:
    containeres:
    - name: simple-webapp
      image: simple-webapp
      ports:
        - containerPort: 8080
   ```
## セレクター
### コマンド
- `kubectl get` コマンドなどで `selector` オプションでラベルを指定する
  ```
  kubectl get pods --selector app=App1
  ```
- ヘッダーを除いて表示する
  ```
  kubectl get pods --selector app=App1 --no-headers
  ```
- 複数のラベルを指定する場合、ラベル間は `,` で区切る
  ```
  kubeclt get all --selector env=prod,bu=finance,tier=frontend
  ```
### 内部的に使用する場合
例：レプリカセット 
- Pod の定義にラベルを付け、レプリカセットでセレクターを使用して Pod をグループ化する
  ```yaml
  apiVersion: v1
  kind: ReplicaSet
  metadata:
    name: asimple-webapp
    labels:  # レプリカセットに設定したラベル
      app: App1
      function: Front-end
  spec:
    replicas: 3
    selector:
      matchLabels:  # レプリカセットと Pod を結びつけるためのラベル
        app: App1
    template:
      metadata:
        name: siple-webapp
        labels:  # Pod に設定したラベル
          app: App1
          function: Front-end
      spec:
        - name: simple-webapp
          image: simple-webapp
          ports:
            - containerPort: 8080
  ```
## アノテーション
- `labels` と同じキー・バリュー形式
- ラベルのようなオブジェクトの識別に使用できない
- オブジェクトのメモ的な情報を記録
  ```yaml
  apiVersion: v1
  kind: ReplicaSet
  metadata:
    name: asimple-webapp
    labels:  # レプリカセットに設定したラベル
      app: App1
      function: Front-end
    annotations:
      buildversion: 1.34
  spec:
    replicas: 3
    selector:
      matchLabels:  # レプリカセットと Pod を結びつけるためのラベル
        app: App1
    template:
      metadata:
        name: siple-webapp
        labels:  # Pod に設定したラベル
          app: App1
          function: Front-end
      spec:
        - name: simple-webapp
          image: simple-webapp
          ports:
            - containerPort: 8080
  ```

# Taints と Tolerations
## 働き
- Pod のノードへのスケジューリングに制限を設ける機能
- 考え方
  1. Pod とノードに Taint （汚れ）を設定する
  2. ノードは Pod のスケジューリング時に Taint を確認し、同じ Taint の場合はスケジューリングを Toleration （容認）し、異なる Taint の場合は InToleration （否認）する
## コマンド
### ノード
- ノードに Taint を設定する
  ```
  kubectl taint nodes node-name key=value:taint-effect
  ```
  ```
  kubectl taint nodes node1 app=blue:NoSchedule
  ```
  - `taint-efect` に設定する値
    - NoSchedule  
      - Taint に一致しない新規 Pod はノードにスケジュールされない
      - ノードに配置済みの Pod はそのままになる
    - PreferNoSchedule
      - Taint に一致しない新規 Pod はノードにスケジュールされる可能性があるが、スケジューラーはスケジュールしないようにする
      - ノードに配置済みの Pod はそのままになる
    - NoExecute
      - Taint に一致しない新規 Pod はノードにスケジュールできない
      - 一致する Taint を持たないノードの既存 Pod は削除される
- ノードから Taint を削除する
  ```
  kubectl taint nodes node-name key=value:NoSchedule-
  ```
  ```
  kubectl taint nodes node1 app=blue:NoSchedule-
  ```
- 
### Pod
- Pod に Taint を設定する
  ```yaml
  apiVersion: v1
  kind: Pod
  metadata:
    name: myapp-pod
  spec:
    containers:
    - name: nginx-container
      image: nginx
    tolerations:
    - key: "app"
      operator: "Equal"
      value: "blue"
      effect: "NoSchedule"
  ```
